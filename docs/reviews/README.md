## Review File Guidelines

### Naming Convention
- Use descriptive names: `COMPONENT_REVIEW.md` or `TOPIC_ANALYSIS.md`
- Include date in filename if multiple versions: `PROJECT_REVIEW_2025-11-11.md`
- Use uppercase with underscores: `BASH_WORKFLOWS_REVIEW.md`

### Content Structure
- **Date** and **Reviewer** at top
- **Executive Summary** with key findings
- **Detailed Analysis** by category
- **Recommendations** prioritized
- **Conclusion** with overall assessment

### Consolidation
- Consolidate related reviews when possible
- Archive outdated reviews instead of deleting
- Keep only current, relevant reviews in main directory
